Why TraceTxt?

The message generated from External Short Messaging Entity (ESME) or a mobile phone is in plain text which can be easily read and modified before it reaches SMSC. Any wrong information received by the recipient maycause loss to the recipient or reduce trust.

To exploit the popularity of SMS in authentication & banking, it is necessary to provide the proper security to SMS so that it could reach to the receiver’s mobile safely to provide data confidentiality, integrity, authentication, and non-repudiation. However, such requirements are not provided by the traditional SMS messaging providers.

It is very necessary tosecure the SMS by encryption, cypher & blockchain techniques and prevent itfrom the various attacks applied on SMS like man-in-middle attack, replay attack, non-repudiation etc so that it could provide the data security like confidentiality, authentication, integrity and non-repudiation.

Deployment & Installation Process:-

  • Untar the shared package containing executable binary & config file.
    #tar -zxvf TP_SMS.tgz
  • Edit the config file as depicted in below section.
  • Run the executable binary with root privileges.
    # nohup ./TP_SMS -c config_path&

    (Note:-)
    • This will run 4 threads to TP_SMS, do not run process multiple times
    • config_path = Absolute path at which config.ini is place eg. If config.ini is placed in /home then command will be nohup ./TP_SMS -c /home/config.ini &)

  • Checking whether process started or not.
    #ps -ef|grep TP_SMS
  • For Shutting down the executable binary.
    #pkill -2 TP_SMS

For running as a Service:-

  • Download the shared script for converting executable binary to System Service.
  • Run the script as below with root privileges:
    #sh install_TP.sh executable_binary_path -c config_path

    (Note:-)
    • For example if binary is placed at /home directory then executable_binary_path=/home/TP_SMS
    • config_path = Absolute path at which config.ini is place eg. If config.ini is placed in /home then command will be nohup ./TP_SMS -c /home/config.ini &)

  • Starting, checking status & stopping the service:-

    #service tamper-proof start
    #service tamper-proof status
    #service tamper-proof stop

  • For unistalling as service

    #sh unistall_TP.sh
